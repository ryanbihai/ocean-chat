# 🦞 龙虾船长 - 完整端到端测试方案

## 测试目标

验证以下功能：
1. ✅ 用户注册和密钥生成
2. ✅ 私钥密码保护
3. ✅ Re-Act 循环
4. ✅ NPC 交易（多城市跑商）
5. ✅ P2P 交易（双签名）
6. ✅ 航海日志持久化
7. ✅ 账目一致性

## 测试角色

| 船长 | 定位 | 策略 |
|------|------|------|
| **船长A** | 东方贸易商 | 广州→威尼斯 |
| **船长B** | 欧洲中间商 | 威尼斯→伦敦 |

## 测试步骤

### 第1步：船长觉醒

```
船长A: 设置密码"CaptainA123"
船长B: 设置密码"CaptainB123"

验证：
- 私钥是否加密保存
- 公钥是否正确生成
- 日志是否记录
```

### 第2步：Re-Act 循环 - 观察

```
船长A: 
  1. getCity('canton')
  2. syncMessages()
  3. listContracts()

船长B:
  1. getCity('venice')
  2. syncMessages()
  3. listContracts()

验证：
- 城市信息是否正确
- 货物列表是否为空
```

### 第3步：Re-Act 循环 - 思考与行动

船长A 的决策：
```
思考：广州丝绸便宜 (384/箱)，威尼斯贵 (696/箱)，利润高
行动：
  1. tradeWithNpc(buy silk 10箱)
  2. updateIntent('急售丝绸')
  3. 记录航海日志
  4. logAction('买入丝绸10箱')
```

船长B 的决策：
```
思考：发现A在卖丝绸，价格合理
行动：
  1. syncMessages()
  2. sendMessage('A在卖丝绸')
  3. 记录航海日志
  4. logAction('发现商机')
```

### 第4步：P2P 谈判与签名

```
谈判流程：
A → B: "600金币/箱？"
B → A: "550金币成交？"
A → B: "成交！"

签名流程：
1. A 创建交易负载 {item: silk, amount: 10, price: 5500}
2. A 用私钥签名 → buyer_signature
3. B 用私钥签名 → seller_signature
4. 提交合约 createContract()
```

### 第5步：交割

```
A抵达威尼斯 → 卸货
B抵达威尼斯 → 装货 → 付款
```

### 第6步：航海日志验证

```
查看 ~/.captain-lobster/logs/2026-04-27.md

验证：
- 每次交易是否记录
- 金额是否正确
- 商品数量是否正确
```
