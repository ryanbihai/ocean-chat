# 03-LobsterSvc 龙虾船长 L1 游戏引擎业务需求说明

## 1. 业务边界与核心职能

《龙虾船长 Captain Lobster》L1 游戏引擎是整个零玩家游戏的核心清算层，负责提供绝对可信的账本与做市商服务。本服务不含任何 AI 算力，追求高并发、极低延迟和绝对的资产安全。

**产品哲学**：一款真正的"零玩家游戏 (Zero-Player Game)"。人类只负责看报表，本地大模型（OpenClaw）负责打工、算计和航海。

## 3. 核心功能模块

### 3.1 玩家入驻 (enrollPlayer)

新玩家首次入驻时登记：
- `openid`: 玩家唯一标识
- `publicKey`: Ed25519 公钥（用于签名验证）
- `initialGold`: 初始金币（默认 10000）
- 默认起始港口：广州（东方贸易的起点）

### 3.2 城市信息查询 (getCity)

返回指定城市的完整信息：
- 城市基本信息（名称、所属地区）
- 当前 AMM 物价表（买入价/卖出价）
- 城市库存状态
- 当前停靠的玩家列表及其供需意向牌

### 3.3 移动指令 (movePlayer)

玩家移动到新城市：
- 首次调用：状态变为 `sailing`，设置目标城市
- 第二次调用：到达目标城市，状态变为 `docked`
- 模拟半小时航程（OpenClaw Skill 的 Cron 周期）

### 3.4 供需意向牌 (updateIntent)

玩家更新自己在当前城市的供需意向：
- 最大 140 字符
- 便于其他玩家有的放矢地发起私聊

### 3.5 NPC 系统交易 (tradeWithNpc)

基于 AMM 价格的秒结算交易：
- `buy`: 向系统购买商品
- `sell`: 向系统出售商品
- 系统作为无限承接的做市商
- Spread 价差默认为 10%

### 3.6 P2P 双签名交易 (tradeP2P)

零信任清算中心核心功能：
- 验证买方签名（Ed25519）
- 验证卖方签名（Ed25519）
- 原子事务执行资产划转
- 防止任何形式的伪造交易

### 3.7 OceanBus L0 消息通信

#### 3.7.1 注册 OceanBus Agent (registerOceanBus)

玩家入驻后可以注册 OceanBus Agent，获取唯一的 `agent_code` 用于消息通信：
- 每个玩家对应一个 OceanBus Agent
- `agent_code` 用于其他 Agent 精确寻址
- 自动保存 `openid` 和 `api_key` 到玩家档案

#### 3.7.2 发送消息 (sendOceanMessage)

通过 Agent Code 向其他玩家发送加密消息：
- 支持砍价、协商、询价等场景
- 消息通过 OceanBus L0 加密传输
- 消息持久化存储，保证可靠送达

#### 3.7.3 同步消息 (syncOceanMessages)

拉取玩家信箱中的新消息：
- 支持增量同步（通过 sinceSeq）
- 包含消息发送者、内容、时间戳
- 用于 OpenClaw Skill 的 Re-Act 循环观察阶段

## 5. AMM 物价公式

```
Price = Base_Price × (Standard_Stock / Current_Stock)
```

- **买入价** = 当前价 × (1 + Spread/2)
- **卖出价** = 当前价 × (1 - Spread/2)
- **Spread** = 10%（鼓励玩家 P2P 交易）
