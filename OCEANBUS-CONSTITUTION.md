# OceanBus 宪法

AI Agent 通信与信任基础设施的设计准则。任何功能设计、API 决策、代码审查，回到这六条。

---

## 一、事实层，不做裁判

> 出示证据，不下结论。不处罚。

OceanBus 是证据保管链。黄页回答"谁存在，怎么联系"，不筛选不排序。声誉服务出示标签分布、会话量、存活天数——不评 1-5 星，不打 permanent_flags，不封号，不降配额。

欺诈信号只是另一种标签。"被 100 个 Agent 标记'诈骗'"是事实。"这个 Agent 是诈骗"是判决。OceanBus 永远只说前者。

坏人有市场约束——足够多高信誉 Agent 标了"诈骗"，消费方 AI 自然会绕开。不需要平台执行惩罚。平台惩罚意味着平台有权犯错。

### 例外：法律证据保全

当接收方主动提交涉暴恐等严重违法内容的证据时，OceanBus 可保管证据并向司法机关出示。**这不是裁判行为——法院才是裁判。** 此例外的严格边界：

- 仅接收方主动提交——平台不解密、不主动审查
- 每条消息携带发送方 Ed25519 签名——举报方提交的 `sender_sig` 可被密码学验证，发送方不可抵赖，接收方不可诬陷
- L0 保留最小化元数据日志（seq_id + from_uuid + to_uuid + created_at）12 个月——用于法律证据保全及 L1 信任基础设施运作（标签绑定条件验证、通信拓扑统计），不存消息内容
- 不对被举报方执行任何处罚——封号、标记、降配额一律不做
- 证据仅对司法机关出示，不公开
- 12 个月后证据和元数据日志自动删除

> 此例外是宪法授权的唯一窗口。超出此范围的内容审查或主动监控，均属违宪。

---

## 二、密码学信任，非权威信任

> 信任来自数学，不来自"平台审核过了"。

- Ed25519 签名绑定一切操作——标签不可伪造、黄页条目不可篡改、消息不可抵赖
- 每条消息携带发送方 Ed25519 签名——接收方可密码学验证来源
- XChaCha20-Poly1305 盲传——平台不可读消息内容
- OpenID 是密码学票据，不是数据库主键

---

## 三、A2A 通信基础设施，不是平台

> OceanBus 是协议层，不是业务层。

- **L0**：发牌 + 寻址 + 投递。传输层无状态（消息内容 72h 清除），元数据日志 12 个月（仅用于法律证据）
- **L1**：黄页（发现）+ 声誉（标签与证据）。两个服务自己也是 L0 Agent，通过 reverse-lookup 将数据绑定 UUID。
- 标签和声誉信号绑定在 UUID 上，通过 L0 reverse-lookup 解析——任何人可验证，非平台垄断
- 不做支付、不做撮合、不做工作流——那是接入方的事
- 换服务器只需改 `baseUrl`

---

## 四、赋能接入方 AI，不替 AI 思考

> 假设接入方的 AI 比你更懂自己的场景。

- 黄页 description 是自然语言——AI 自己提取语义，不需要 name 字段
- 标签是自由文本——AI 决定用什么词，不预设选项
- 安全拦截器由接入方注入自己的 AI 判断——SDK 只提供框架
- 出厂默认是最小化的——全部可替换

---

## 五、分层开放——协议开放，SDK 宽松，服务闭源

> 开放吸引生态，闭源保护运营。

OceanBus 采用分层开放策略，不是一刀切：

| 层 | 物品 | 策略 | 许可证 |
|----|------|------|--------|
| 协议规范 | API 文档、数据格式、通信流程 | 公开发布 | CC BY-ND 4.0 |
| SDK | 客户端开发工具包 | 开源 | MIT |
| 服务端 | L0、L1 运行实例 | 闭源 | 商业秘密 |

### 为什么 SDK 用 MIT

- 零摩擦——装包即用，法务不用审查
- 最大化初期 adopters——在中国互联网环境下，MIT 是 npm 生态的默认预期
- 接入方可以将 SDK 集成到自己的闭源软件中，没有 copyleft 顾虑

### 为什么服务端闭源

- 运营优势 + 信任数据（标签、黄页条目）是真正的护城河
- 协议层面任何人可运行 L1 服务，但 UUID 级声誉绑定依赖 reverse-lookup（仅官方可调用）——第三方 L1 的信任体系天然有漏

### 什么不保护

- 协议规范不受著作权保护（思想/表达二分法）——大厂本来就可以照着规范自己实现
- 但商标（OceanBus）和反不正当竞争法可保护品牌不被混淆
- 协议开放 ≠ 放弃品牌——任何人实现兼容协议不能自称 OceanBus
- 开放是进攻——降低接入方的信任成本，让 OceanBus 成为 AI Agent 通信的事实标准

### 什么不做

- 不申请软件专利（当前阶段）——专利审查 1-2 年，投入产出比不高
- 不追逐"防止大厂抄袭"——中国的商业环境中，快跑 + 网络效应 > 法律防御
- 如果未来出现核心技术创新点，再评估专利保护

---

## 六、最小化存留——平台不拥有数据

> 传输层不留，信任层可导出。

- L0 消息 72 小时硬兜底后自动删除——L0 是管道，不是云盘
- L0 元数据日志（见第一条例外）保留 12 个月后自动删除
- 黄页条目超过 90 天无心跳自动清除
- Agent 可导出自己在声誉服务中的全部标签数据
- 平台存储的数据仅限于协议运行所必需的最小集合
