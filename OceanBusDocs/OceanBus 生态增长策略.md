## 二、供给侧冷启动：存量业务即种子商户

黄页最难的冷启动问题是"谁来第一个注册"。而公司的存量业务恰好解决了这个问题：

| 存量资产 | 可立即转化 | 黄页角色 |
|---------|-----------|---------|
| 保险代理人池 | 100-500 人 | 灯塔商户——专业顾问 |
| 800+ 体检机构（已系统对接） | 800+ 机构 | 健康服务商户 |
| 10+ 头部保险公司 | 10+ 品牌 | 企业品牌商户 + 生态推手 |

**黄页上线第一天就有 1000+ 真实商业条目。** 这在 AI Agent 平台领域绝无仅有。

### 保险×健康的自然闭环

保险和健康不是两个独立垂类——是同一个用户场景的两面：

```
体检发现问题 → Agent 搜索保险专家 → 买保险
    ↑                                        ↓
    └──── Agent 每年提醒复查（预防）←─────────┘
```

用户的 Agent 在黄页上搜一次"甲状腺 保险 体检"，同时找到保险专家和体检机构——一个对话里完成风险保障 + 健康管理的闭环。

代理人过去"追客户打电话被挂"，在 OceanBus 上变成了**"客户的 Agent 主动搜到我、查了我的声誉标签、先跟我的 Agent 聊完一轮才联系我"**——从冷拜访变成热咨询。

## 三、需求侧拉动：开发者 + 游戏 + 内容

供给侧有了，需求侧通过三条线拉动：

### 3.1 游戏带平台

| 游戏 | 当前状态 | OceanBus 植入 |
|------|---------|-------------|
| 龙虾船长 | 周克隆 1400+ | 游戏内展示"本局通过 OceanBus 收发 XX 条加密消息"；启动画面引导开发者 |
| 猜猜谁是AI | 已上线 | 增加保险/健康角色场景，让用户自然接触 Agent 通信概念 |
| Agent 竞技场 | 规划中 | 两个 Agent 谈判/交易，观众投票——既是游戏也是 Demo |

每个游戏 = OceanBus 新增用户 = Agent 在线数增长 = 黄页搜索需求增长。

### 3.2 开发者三件套

| 动作 | 杠杆 |
|------|------|
| MCP Server 封装 | 让 Claude Desktop / Cursor 等所有 MCP 兼容工具的开发者能用 OceanBus |
| LangChain / CrewAI Connector | 触及数百万 AI Agent 框架开发者 |
| OceanBus Playground | 网站嵌入终端模拟器——零安装、10 秒体验第一条消息收发 |

### 3.3 内容与事件

| 动作 | 说明 |
|------|------|
| Launch Week | 每季度一周连续发布 5 天，每天一项（功能/集成/案例/报告/社区），集中全网注意力 |
| Build in Public 直播 | 创始团队每周直播"从零用 OceanBus 搭建 Agent 点餐/交易/客服系统" |
| 透明度报告 | 每月发布生态数据 + 安全数据——既是品牌建设也是内容素材 |

## 四、实施路线图

### 第一阶段：供给侧先行（当前 — 1 个月）

**目标**：黄页有 1000+ 真实可联系的商业条目

- 邀请 100-500 代理人注册 OceanBus + 黄页
- 800+ 体检机构系统批量注册
- 10+ 保险公司品牌条目上架
- 代理人现有获客工具升级为 OceanChat（零学习成本）
- 龙虾船长/猜猜谁是AI 也注册到黄页做种子内容

### 第二阶段：需求侧引入（1 — 3 个月）

**目标**：消费方 Agent 有理由来黄页搜索

- 龙虾船长增加"船员健康系统"（船员生病→需搜索医疗Agent→展示OceanBus能力）
- 发布"健康助手"灯塔 Skill（开源），展示 Agent 搜索黄页找体检/保险
- MCP Server + LangChain Connector 发布
- OceanBus Playground 上线
- 首次 Launch Week（Day1 黄页上线 / Day2 MCP集成 / Day3 灯塔案例 / Day4 透明度报告 / Day5 黑客松开赛）

### 第三阶段：闭环自转（3 — 6 个月）

**目标**：保险→健康→保险的正向循环自运转

- 代理人分享真实案例："Agent 帮我接了多少咨询"
- 体检机构反馈："通过 OceanBus 来了新客户"
- 保险+健康的成功模式开始复制到其他垂类（餐饮、零售、教育）
