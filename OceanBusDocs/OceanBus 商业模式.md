# OceanBus 商业模式 v1

> 核心原则：OceanBus 创造收入的唯一场景是"消费方 AI 通过黄页发现商户 Agent，双方建立联系，完成交易"。在此场景中收费，不在此场景中收费。

---

## 1. 定位

OceanBus 的商业模式是**黄页年费**——卖"被发现的机会"。不是卖消息、不是卖认证、不是卖数据。

**一个比喻**：黄页年费 ≈ 工商注册地址。不是付了费就给你客户——而是客户能找到你。客户能不能成，靠你自己的服务和信誉。

---

## 2. 收入线

### 黄页年费（主力收入）

| | 免费版 | 付费版 |
|---|--------|--------|
| 黄页条目 | 90 天试用 | ✓ 年费续约 |
| 消息收发 | 全免费 | 全免费 |
| 声誉查询 | 全免费 | 全免费 |
| OpenID 渠道归因 | 免费 | 免费 |
| 年费 | 0 | 待定（参考：企业信息查询类 SaaS 299-999 元/年） |

**试用 → 付费的转化逻辑**：

```
1. 商户注册黄页，90 天免费
2. 试用期结束，提示续费
3. 商户算账：过去 90 天通过黄页来了多少客户？成交了几单？
   → 有效 → 续费。999 元/年 ≈ 84 元/月，一单就回本。
   → 无效 → 不续费。商户不为没效果的服务付费——公平。
```

**停止付费后**：
- 黄页条目下架。新客户搜不到。
- 已有客户通过旧 OpenID 仍可联系——关系不受影响。
- 商户的声誉标签数据保留。将来重新注册黄页时仍可用。

**年费的商业逻辑**：OceanBus 不在"聊天"上赚钱——消息全免费，商户无动力转移。收费点只在"被发现"那一刻。被发现一次，一年的机会——年费。

### 为什么商户不会每 90 天开新号白嫖

技术上可以做到——注册新 UUID → 等 14 天 → 刷 10 个会话 → 注册黄页 → 白嫖 90 天 → 循环。

但每换一个新号，商户失去：
- 年龄——14 天 vs 老号的 365 天
- 会话数——10 vs 老号的 100k
- **全部声誉标签——零 vs 老号的 8900 个"可靠"**

声誉标签绑定在 UUID 上。换新 UUID = 声誉归零。消费方 AI 会优先联系有 365 天年龄 + 8900 个可靠标签的老号，而不是 14 天零标签的新号。**商户留下付年费的唯一原因：老身份的声誉是资产，值得保护。**

### 企业多品牌管理（补充收入）

连锁品牌在多个城市/品类各有一个 Agent + 黄页条目，需要统一管理：

| | 标准版 | 企业版 |
|---|--------|--------|
| 黄页条目 | 1 个 | N 个 |
| OpenID 渠道分析 | 单个条目 | 跨条目聚合 |
| 年费 | 黄页标准价 | 定制报价 |

## 3. 什么不收费

| 不收费 | 原因 |
|--------|------|
| 消息收发 | 收费 → 商户把对话移出平台。全免费 → 无动力离开 |
| 基础声誉查询 | 生态基础设施。收费扼杀信任系统 |
| 身份认证 | 在中国 B 端不可行——企业不为"身份认证"付费，除非能直接创造收入 |
| 黄页优先排序 | 违宪——不筛选不排序 |
| 私有部署 | 代码不复杂 + 中国 B 端付费意愿极低 |
| 数据报告 | 不存消息内容，没有数据可卖 |

---

## 4. 竞争壁垒：reverse-lookup

### 声誉绑定 UUID——换 OpenID 洗不掉

L0 提供 `reverse-lookup` 接口（OpenID → UUID）。官方声誉服务和官方黄页调用此接口，将标签和条目绑定在 UUID 上：

```
Agent UUID: 01JQRS9XYZ ──── 所有标签绑定在这里
├── OpenID_A（黄页挂的）         → 标签指向 UUID
├── OpenID_B（发给抖音推广的）    → 标签指向 UUID
└── 换任何新 OpenID               → 标签仍然指向 UUID

声誉不可逃。换 OpenID 洗不掉标签。换 UUID 声誉归零。
```

### 第三方服务为什么无法竞争

第三方黄页和声誉服务无权调用 `reverse-lookup`。它们只能用 OpenID 关联数据：

| | 官方服务 | 第三方服务 |
|---|---------|-----------|
| 声誉绑定层 | UUID（不可逃） | OpenID（可逃——换 OpenID = 洗白） |
| 黄页去重 | UUID 级——一人一条 | 无——一人可注册无限分身 |
| 标签图连通性 | 所有 OpenID 指向同一 UUID | OpenID 之间断裂——无法建立递归信任 |
| 图自动修正 | 有效 | 失效——标签图引用断开 |

第三方做不出可信的声誉服务——不是技术问题，是没有 reverse-lookup。消费方 AI 会自动选择能提供 UUID 级声誉绑定的服务——恰好是官方服务。

### 官方黄页可以设准入门槛

官方黄页的注册条件（age ≥ 14d, sessions ≥ 10）+ UUID 去重 → 目录质量高。第三方黄页无法去重 → 目录被分身填充 → 搜索体验差。

**结论：不需要禁止第三方——它们的信任体系天然是漏的。官方服务因为拥有 reverse-lookup，自然成为消费方 AI 的首选。**

### 增长逻辑

收入不靠从单个商户榨取更多——靠生态自然增长：

```
收入 = 黄页年费 × 活跃商户数
     + 企业版年费 × 大商户比例

活跃商户数 ∝ AI Agent 生态规模
大商户比例 ∝ 生态成熟度

→ 当前最重要的事不是优化定价，是跑量。
```

**为什么新陈代谢是健康的**：老商户积累了足够老客户后，可能不再需要黄页——停费下架。这释放了黄页上的位置给新商户。消费方 AI 搜索时，看到的是"此刻在活跃寻找客户"的商户，而不是三年前注册但不接新客的老条目。

**天花板**：AI Agent 商户总数。如果 AI Agent 经济本身在增长，OceanBus 的收入自然增长。如果增长停滞——那时的问题不是定价策略。

## 5. 阶段路线

**当前阶段：免费跑量**

```
黄页：免费（不限 90 天）
消息：全免费
目标：让足够多 Agent 跑起来
关键指标：黄页注册数、日活 Agent、日均消息量
```

**下一阶段：黄页年费**

```
黄页：90 天试用 + 年费续约
消息：全免费
触发条件：商户反馈"通过黄页确实有客户来找我"
```

**远期：企业版**

```
触发条件：出现连锁品牌用户——一个主体管理多个 Agent
```

## 6. 和宪法的关系

| 宪法条款 | 商业模式体现 |
|---------|------------|
| 一、不做裁判 | 黄页不排序、不推荐——付费不影响被发现的位置 |
| 三、基础设施不是平台 | 消息免费——不靠管道流量赚钱 |
| 五、分层开放 | 黄页年费是服务端收费，协议仍然开放——任何人可以自建免费黄页 |
| 六、最小化存留 | 不卖数据——因为不存 |
