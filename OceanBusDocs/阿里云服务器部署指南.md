## 一、首次部署

### 1.1 环境准备

```bash
# 安装 Node.js（如未安装）
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 PM2
npm install -g pm2

# 安装 git
sudo apt-get install -y git
```

### 1.2 克隆仓库

```bash
cd ~
git clone https://github.com/你的仓库地址.git 项目目录名
cd 项目目录名
npm install
```

### 1.3 配置 PM2

在项目根目录创建 `pm2-start.json`：

```json
{
  "apps": [
    {
      "name": "你的服务名",
      "script": "src/server.js",
      "instances": 1,
      "exec_mode": "fork",
      "env": {
        "NODE_ENV": "production",
        "PORT": 17020
      }
    }
  ]
}
```

### 1.4 启动服务

```bash
# 首次启动
pm2 start pm2-start.json

# 保存 PM2 进程列表（服务器重启后自动恢复）
pm2 save

# 设置开机自启
pm2 startup
# 按提示执行输出的 sudo 命令
```

## 三、常用 PM2 命令

```bash
pm2 list              # 查看所有服务状态
pm2 logs 服务名        # 查看日志（实时）
pm2 logs 服务名 --lines 100  # 最近 100 行
pm2 restart 服务名     # 重启
pm2 stop 服务名        # 停止
pm2 delete 服务名      # 删除（重新部署前）
pm2 monit             # 实时监控面板
pm2 save              # 保存当前进程列表
```

## 四、GitHub 访问问题

阿里云 ECS 访问 GitHub 可能遇到 TLS 握手失败：

```
fatal: unable to access 'https://github.com/...': GnuTLS recv error (-110)
```

**临时方案**（降级为 HTTP）：

```bash
git config --global url."http://github.com/".insteadOf "https://github.com/"
git pull
# 拉完改回去！
git config --global --unset url."http://github.com/".insteadOf "https://github.com/"
```

**持久方案**（SSH 免密）：

```bash
# 生成 SSH 密钥
ssh-keygen -t ed25519 -C "admin@aliyun"

# 复制公钥
cat ~/.ssh/id_ed25519.pub
# → 添加到 GitHub → Settings → SSH and GPG keys

# 切换 remote
git remote set-url origin git@github.com:用户名/仓库名.git
```

## 六、日志与监控

```bash
# PM2 日志路径
~/.pm2/logs/

# 查看磁盘使用
df -h

# 查看内存
free -h

pm2 install pm2-logrotate
pm2 set pm2-logrotate:max_size 10M
pm2 set pm2-logrotate:retain 7
```

## 七、Nginx 反向代理（如需对外提供 HTTP）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:17020;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```bash
# 测试配置
sudo nginx -t

# 重载
sudo nginx -s reload
```
